Smart commit for Countrivo.

1. Run `npm run build` to verify no type/build errors. If it fails, fix the issues first.
2. Run `git status` and `git diff --staged` to review changes.
3. Analyze what changed and draft a concise, narrative commit message matching the project style:
   - Feature-focused, descriptive (e.g., "Add Blitz game: flag speed recognition with multiplayer support")
   - No conventional-commits prefixes (no feat:, fix:, chore:)
4. Stage relevant files. Never stage .env*, *.pem, or node_modules.
5. Commit with the drafted message.
6. Show the commit hash and summary.
